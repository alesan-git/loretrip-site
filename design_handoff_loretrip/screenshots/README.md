# Drop user-provided app screenshots in this folder.

When LoreTrip provides product screenshots, save them here with these exact filenames so the marketing site picks them up automatically:

| Filename | What it is | Used in |
|---|---|---|
| `dashboard.png` | Agency admin dashboard | Landing "How it works" row 1, Brand book section 07 |
| `planner.png` | Trip planner / itinerary builder | Landing "How it works" row 3 |
| `supplier-network.png` | Supplier network UI | Landing "How it works" row 2 |
| `client-view.png` | Client-facing branded planner (under agency's domain) | Hero right-side image |
| `extras/*.png` | Any additional screenshots | Brand book section 06 (Visual Language) — replace `[ PLACEHOLDER · COMMISSION ]` cards |

## Image requirements

- 2× resolution (Retina) — minimum 2560px wide for full-viewport hero shots
- PNG with transparent or `#FBFAF6` (Surface) background where possible
- Trim browser chrome, OS chrome, and any irrelevant UI — the screenshot should be just the app surface
- Light theme preferred (matches brand palette without re-coloring)

If a screenshot is missing at build time, the placeholder card stays in. **Never substitute stock imagery for missing screenshots.**
